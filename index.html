HTTP协议学习笔记

简介

HTTP协议使用在TCP/IP协议的基础上，应用请求/响应的通信模式，实现网页、图片、视频等各种网络数据的传输。

工作原理

HTTP工作在Clent-Server架构上，作为Client的浏览器使用URL的方式向作为Server的服务器发送Request请求，服务器返回给浏览器Response作为回应。
HTTP工作有3个特点：

  1. 每次连接只处理1个请求，服务器处理完请求，并收到浏览器的应答后就断开服务。
  2. 可以处理任意的数据内容，使用MIME-type来标识内容类型，也称为content type。
  3. 每次响应之间不保留状态。

消息格式

每个HTTP的请求和回应都遵循相同的消息格式，由Header和Body二部分组成，使用一个空行来分隔。

请求的格式

Header

1-4行为Header:

  • POST为请求的方法，一般为GET和POST等，共8个。
  • /path为URL，标识访问的资源。
  • HTTP/1.1为协议版本号，指明使用HTTP1.1版。
  • Header1: Value1等为头部属性，每个属性独占1行。

Body

6行开始，为需要传输的数据。

响应的格式

Header

1-4行为Header:

  • 200 OK为响应状态行，标识服务器对请求的处理结果。
  • Header1: Value1等为头部属性，每个属性独占1行。

Body

6行开始，为需要传输的数据。

8种请求方法

HTTP1.0定义了三种请求方法： GET, POST 和 HEAD方法。
HTTP1.1新增了五种请求方法：OPTIONS, PUT, DELETE, TRACE 和 CONNECT 方法。

序号 方法 描述
1 GET 请求指定的页面信息，并返回实体主体。
2 HEAD 类似于get请求，只不过返回的响应中没有具体的内容，用于获取报头
3 POST 向指定资源提交数据进行处理请求（例如提交表单或者上传文件）。数据被包含在请求体中。POST请求可能会导致新的资源的建立和/或已有资源的修改。
4 PUT 从客户端向服务器传送的数据取代指定的文档的内容。
5 DELETE 请求服务器删除指定的页面。
6 CONNECT HTTP/1.1协议中预留给能够将连接改为管道方式的代理服务器。
7 OPTIONS 允许客户端查看服务器的性能。
8 TRACE 回显服务器收到的请求，主要用于测试或诊断。

响应的状态码

常见的HTTP状态码：

  • 200 请求成功
  • 301 资源（网页等）被永久转移到其它URL
  • 404 请求的资源（网页等）不存在
  • 500 内部服务器错误

HTTP状态码分类

HTTP状态码由三个十进制数字组成，第一个十进制数字定义了状态码的类型，后两个数字没有分类的作用。HTTP状态码共分为5种类型：

分类 分类描述
1** 信息，服务器收到请求，需要请求者继续执行操作
2** 成功，操作被成功接收并处理
3** 重定向，需要进一步的操作以完成请求
4** 客户端错误，请求包含语法错误或无法完成请求
5** 服务器错误，服务器在处理请求的过程中发生了错误

头部属性

提供了关于请求，响应或者其他的发送实体的信息。

  • Allow
    服务器支持哪些请求方法（如GET、POST等）。
  • Content-Encoding
    文档的编码（Encode）方法。只有在解码之后才可以得到Content-Type头指定的内容类型。利用gzip压缩文档能够显著地减少HTML文档的下载时间。
  • Content-Length
    表示内容长度。只有当浏览器使用持久HTTP连接时才需要这个数据。
  • Content-Type
    表示后面的文档属于什么MIME类型。默认为text/plain，但通常需要显式地指定为text/html。
  • Date
    当前的GMT时间。可以用setDateHeader来设置这个头以避免转换时间格式的麻烦。
  • Expires
    应该在什么时候认为文档已经过期，从而不再缓存它？
  • Last-Modified
    文档的最后改动时间。
  • Location
    表示客户应当到哪里去提取文档。
  • Refresh
    表示浏览器应该在多少时间之后刷新文档，以秒计。
  • Server
    服务器名字
  • Servlet
    一般不设置这个值，而是由Web服务器自己设置。
  • Set-Cookie
    设置和页面关联的Cookie。
  • WWW-Authenticate
    客户应该在Authorization头中提供什么类型的授权信息。
